原论文中的 step entropy 定义不是 top-k entropy，而是 全词表分布熵。

$H(t_{i,j} | c_{i,j}) = - sum_{w \in V} p(w | c_{i,j}) log_2 p(w | c_{i,j})$

这里的 V 是整个 vocabulary，也就是模型词表里的所有 token。要算这个值，需要拿到模型在每个生成位置对整个词表的概率分布，也就是 logits/softmax 后的全词表概率。然后一个 step 的熵是：

$H(S_i | S_{<i}) = sum_j H(t_{i,j} | c_{i,j})$

**论文理论上用的是 完整 token 概率分布**

## 节点生成token的近似熵的计算

### 代码采用了工程近似：

API 返回 top_logprobs -> 只拿 top-k 候选 token 的概率 -> 代码计算近似熵

原因是 OpenAI-compatible API 通常不会返回完整词表 logits/probabilities，只返回：
• 生成出来的 token 的 logprob
• 可选的 top_logprobs，即每一步概率最高的前 k 个候选 token

top-k近似熵：

$H_{top-k}(t) = - sum_{w \in TopK(t)} p(w | c) log_2 p(w | c)$

其中：
•$TopK(t)$：模型在当前生成位置返回的概率最高的 k 个候选 token。
•$p(w | c)$：候选 token w 在当前上下文 c 下的概率。
•代码里 API 返回的是 logprob，所以先做：$p(w | c) = exp(logprob(w))$
然后计算：$entropy = - sum(prob * log_2(prob))$​

### 如果一个 Thought/节点生成了多个 token，代码会得到：

token_entropies_bits $= [H_{top-k}(t_1), H_{top-k}(t_2), ..., H_{top-k}(t_M)]$

sum_entropy_bits = $sum_j H_{top-k}(t_j)$​

上述便是节点的token熵，对应原论文的步骤熵

avg_entropy_bits = $(1 / M) * sum_j H_{top-k}(t_j)$​​ 这是长度归一化的平均熵

avg_neg_logprob_bits = $- (1 / M) * sum_j log_2 p(t_j | c_j)$​
也就是实际生成 token 的平均负 log 概率，单位是 bit。它不是 entropy，只是模型对“实际选中 token”的平均不确定性指标。

可以考虑引入top-k的质量，判断近似程度。如果概率在0.95，说明近似程度好，否则需要进一步考量。

topk_prob_mass = $sum_{w \in TopK} p(w | c)$​

定义节点的生成熵 $H_{gen}(v) = avg\_entropy\_bits$

运行静态 skip 实验时，如果想用归一化后的 API top-k 熵：
python examples/static_skip_experiment.py --data-id 0 --model-name chatgpt --skip-ratio 0.2 --entropy-field normalized_avg_entropy_bits
如果想用原始 top-k 熵：
python examples/static_skip_experiment.py --data-id 0 --model-name chatgpt --skip-ratio 0.2 --entropy-field avg_entropy_bits



你现在可以按下面流程用本地模型测试。
1. 激活环境
conda activate skip-entropy-got
先确认依赖是否有：
python -c "import torch, transformers; print(torch.__version__); print(transformers.__version__)"
如果你用 4bit 加载 32B，还需要：
python -c "import bitsandbytes"
如果报错，说明当前环境还不能 4bit 加载本地 32B。
2. 在 config.json 添加本地模型配置
编辑：
D:/ziliao/skip-entropy-got/graph_of_thoughts/language_models/config.json
加入类似配置：
"deepseek-32b-hf": {
    "model_id": "deepseek-ai/DeepSeek-R1-Distill-Qwen-32B",
    "cache_dir": "",
    "prompt_token_cost": 0.0,
    "response_token_cost": 0.0,
    "temperature": 1.0,
    "top_p": 1.0,
    "top_k": 0,
    "do_sample": true,
    "max_new_tokens": 2048,
    "torch_dtype": "bfloat16",
    "device_map": "auto",
    "load_in_4bit": true,
    "force_single_response_calls": true
}
如果你已经把模型下载到了本地，例如：
D:/models/DeepSeek-R1-Distill-Qwen-32B
那就把 model_id 改成：
"model_id": "D:/models/DeepSeek-R1-Distill-Qwen-32B"
3. 先做一个最小模型加载测试
python -c "from graph_of_thoughts.language_models.factory import build_language_model; lm=build_language_model('graph_of_thoughts/language_models/config.json','deepseek-32b-hf',cache=False); print(type(lm)); print(lm.model_id)"
如果这里能成功，说明模型后端能加载。
4. 跑静态 skip 实验
使用完整词表熵时，entropy-field 仍然用：
avg_entropy_bits
因为本地后端里这个字段已经是完整 vocab 熵，不是 API top-k 熵。
python examples/static_skip_experiment.py --data-id 0 --model-name deepseek-32b-hf --skip-ratio 0.2 --entropy-field avg_entropy_bits
输出会在：
D:/ziliao/skip-entropy-got/examples/static_skip_results/
里面会有：
full_graph.json
compressed_graph.json
summary.json
你重点看 summary.json：
"full": {
    "solved": ...,
    "prompt_tokens": ...,
    "completion_tokens": ...,
    "total_tokens": ...,
    "api_calls": ...,
    "total_latency_seconds": ...
},
"compressed": {
    ...
},
"reductions": {
    "total_token_reduction": ...,
    "api_call_reduction": ...,
    "latency_reduction": ...
}
5. 判断是否真的拿到了完整词表熵
打开 full_graph.json，看某个 thought 的 thought_metadata，应该有：
"entropy_estimate": "full_vocab",
"vocab_size": 152064,
"token_entropies_bits": [...],
"sum_entropy_bits": ...,
"avg_entropy_bits": ...
如果看到：
"entropy_estimate": "full_vocab"
就说明用的是本地模型完整词表熵。
关键注意
为了保证是完整词表熵，配置里保持：
"top_p": 1.0,
"top_k": 0
不要用：
"top_p": 0.95
也不要用：
"top_k": 50
否则生成时的分布可能被截断，算出来就不是完整词表熵，而是截断采样分布的熵。
